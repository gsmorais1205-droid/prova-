Questão 1Python • Imutabilidade
Em Python, pense em mutabilidade como a possibilidade de alterar o conteúdo do mesmo objeto em memória. Um jeito prático de investigar é tentar uma operação mínima e observar o erro (ou ausência dele). 
texto = "abc"
# texto[0] = "A"  # o que acontece?

lista = ["a", "b", "c"]
# lista[0] = "A"  # e aqui?
Se a operação não é suportada, o Python normalmente acusa isso com uma exceção clara.

Qual alternativa é verdadeira?

Questão 2Python • Escopo (LEGB)
Para resolver questões de escopo, não “adivinhe”: localize onde o nome é atribuído e aplique LEGB (Local → Enclosing → Global → Built-in). Atribuição dentro da função cria/usa o nome no escopo local, a menos que você declare global. 
x = "global"

def f():
    x = "local"
    return x

print(f())
print(x)

Qual valor será impresso na segunda linha (print(x))?

Questão 3Python • Exceções
Um bom “resolvedor” segue o fluxo do programa: o que executa no try, o que executa no except. Se a conversão falha, o interpretador salta direto para o bloco de tratamento. 
try:
    n = int("10a")
except ValueError:
    n = 0

print(n)

Questão 4Python • Comprehension
Para não errar comprehension, separe mentalmente em dois passos: 1) filtrar pelo if → 2) transformar com a expressão. 
nums = [1, 2, 3, 4, 5, 6]
pares_quadrado = [n*n for n in nums if n % 2 == 0]
print(pares_quadrado)

Qual será a saída?

Questão 5Python • dict.get
Quando uma chave pode não existir, você escolhe entre: user["email"] (explode com KeyError) vs user.get("email", default) (retorna um padrão). 
user = {"nome": "Ana"}
print(user.get("email", "(sem email)"))
